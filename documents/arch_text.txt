VUE D&apos;ENSEMBLE DE L&apos;ARCHITECTURE                              ▸  1.1  Philosophie &amp; Contraintes                 L&apos;architecture de SÉRÉNOVA est conçue autour de trois impératifs fondamentaux :                               Distribution sans store : génération d&apos;un APK Android installable directement + accès via navigateur web — aucune dépendance à Google Play ou à l&apos;App Store Apple.                      Codebase unique : le même code source produit l&apos;application Android ET l&apos;application web, minimisant la maintenance.                      Scalabilité progressive : conçue pour 10–30 unités, mais l&apos;architecture ne pose aucun plafond technique — extensible à des centaines d&apos;unités sans refonte.                              ▸ Aucune donnée n&apos;est accessible sans authentification valide :                               users : chaque utilisateur ne voit que son propre profil                      sites : SELECT/INSERT/UPDATE/DELETE uniquement sur les sites dont user_id = auth.uid()                      espaces, loyers : accès via la relation sites → user_id = auth.uid()                      locataires : accès si au moins un bail actif lie le locataire à un espace de l&apos;utilisateur                      paiements, alertes : accès via baux → espaces → sites → user_id = auth.uid()                          Aucun accès direct à la base de données n&apos;est possible depuis l&apos;extérieur — toutes les requêtes passent par le SDK Supabase authentifié.                         4.
